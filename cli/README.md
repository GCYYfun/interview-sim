# CLI 控制台应用

使用 **Rich** 库创建的现代化控制台交互应用。

## 🎨 特性

- ✨ 美观的 Rich 界面
- 🎯 交互式菜单系统
- 📊 表格和面板显示
- 🎨 彩色输出和进度提示
- ⌨️ 友好的用户输入提示

## 📁 文件结构

```
cli/
├── app.py           # 主应用入口
├── menus.py         # Rich 菜单系统
├── __init__.py      # 模块导出
└── README.md        # 本文档
```

### 1. 面试模拟 (Interview Simulation)

运行完整的面试流程：

- **模式选择**: 
  - `auto` - AI 自动对话模式
  - `manual` - 人工手动输入模式
  
- **参数配置**:
  - 简历文件路径
  - JD 文件路径（可选）
  - 最大面试轮次

**示例流程**:
```
选择面试模式: auto
简历文件路径: temp/demo_data.csv
JD文件路径（可选）: [Enter跳过]
最大面试轮次: 5

✓ 面试模拟完成！
结果已保存: temp/interview_result_20231016.json
```

### 2. 数据分析 (Data Analysis)

分析和可视化面试数据：

```
选项    功能
────────────────────────────────────────────────────
1       📈 显示统计信息
2       🔍 搜索候选人
3       📊 生成可视化图表
4       💾 导出摘要报告
0       🔙 返回主菜单
```

**功能详解**:
- **统计信息**: 总记录数、岗位分布、平均分数等
- **搜索候选人**: 按岗位、姓名等条件搜索
- **可视化**: 生成各类统计图表
- **导出摘要**: 导出 JSON 格式摘要报告

### 3. 经验提取 (Experience Extraction)

从面试记录中提取和整合经验：

```
选项    功能
────────────────────────────────────────────────────
1       📝 从面试记录提取经验
2       🔗 整合所有经验
3       👁️  查看经验列表
4       💾 导出经验
0       🔙 返回主菜单
```

**使用场景**:
- 从多次面试中总结通用经验
- 整合历史经验形成知识库
- 导出经验用于培训

### 4. 面试助手 (Interview Assistant)

面试准备和辅助工具：

```
选项    功能
────────────────────────────────────────────────────
1       ❓ 生成面试问题
2       🎯 分析简历和JD匹配度
3       📋 完整面试准备
4       📜 从历史记录加载
0       🔙 返回主菜单
```

**功能说明**:
- **生成问题**: 基于简历和 JD 自动生成面试问题
- **匹配度分析**: 分析候选人与岗位的匹配程度
- **完整准备**: 生成完整的面试方案
- **历史加载**: 从历史面试记录加载数据

### 5. 报告查看 (Report Viewer)

查看和管理各类报告文件：

```
选项    功能
────────────────────────────────────────────────────
1       📑 列出所有报告
2       👁️  查看报告详情
3       🔍 搜索报告
4       💾 导出报告
0       🔙 返回主菜单
```

**支持的报告类型**:
- `interview` - 面试结果报告
- `experience` - 经验提取报告
- `plan` - 面试方案报告

**导出格式**:
- `txt` - 纯文本
- `md` - Markdown
- `json` - JSON（保持原格式）

## ⚙️ 配置

### 默认路径设置

在 `cli/app.py` 中可以修改默认路径：

```python
class InterviewSimApp:
    def __init__(self):
        # 默认路径
        self.default_csv = "interview_data.csv"
        self.default_exp_dir = "experiences/"
```

### 延迟加载

所有功能模块采用延迟加载策略，只在需要时才初始化，提升启动速度：

```python
@property
def simulator(self):
    """延迟初始化面试模拟器"""
    if self._simulator is None:
        self._simulator = InterviewSimulator()
    return self._simulator
```

## 🔧 开发指南

### 添加新功能

1. **在 modules/ 中实现功能模块**
2. **在 cli/menus.py 中添加菜单项**
3. **在 cli/app.py 中添加处理函数**

示例：

```python
# cli/menus.py
def show_new_feature_menu(self) -> str:
    """新功能菜单"""
    # ... 菜单代码

# cli/app.py
def handle_new_feature(self):
    """处理新功能"""
    # ... 处理逻辑
```

### 统一错误处理

所有菜单处理函数都使用统一的错误处理模式：

```python
def handle_xxx(self):
    try:
        # 功能逻辑
        pass
    except Exception as e:
        self.menu.show_error(f"操作失败: {str(e)}")
    finally:
        self.menu.pause()
```
